第一次线下赛准备

发表于 2017-11-27 |

2017年11月18日的上海华东杯DCTF是我真正意义上的第一次线下赛，虽然在XMAN结营仪式上大佬也带着我打了一次线下赛，但是因为有大佬带着压力不是很大，而这一次自己单打独斗，所以做了很多的准备。

修改密码

1.修改后台密码：

后台地址：/admin 或者为admin.php

弱口令尝试：admin/admin admin/12345

改成强密码

（如果手速快的话）得知别人的ip地址之后，可以修改别人的后台密码




假设别人手速快，已经改了你的后台密码，可以通过查看配置文件，查看数据库登录账号和密码修改查看新修改的密码，如果这个方法也不行..可能要找主办方重置服务器了

2.修改ssh密码

passwd //修改SSH密码

挂waf

如果是cms，直接在index.php 中require_once（‘waf.php’）;

如果不是框架，在config.php中requir_once('waf.php')；

备份

备份服务器

##RIPS之后手动patch一些字符和字符串(flag,curl) ##
str_replace('flag','','$变量')；

str_replace('curl','','$变量')；

Linux命令

ssh <-p 端口> 用户名@IP　　
scp 文件路径  用户名@IP:存放路径　　　　
tar -zcvf web.tar.gz /var/www/html/　　
w 　　　　
pkill -kill -t <用户tty>　　 　　
ps aux | grep pid或者进程名　　　　
#查看已建立的网络连接及进程
netstat -antulp | grep EST
#查看指定端口被哪个进程占用
lsof -i:端口号 或者 netstat -tunlp|grep 端口号
#结束进程命令
kill PID
killall <进程名>　　
kill - <PID>　　
#封杀某个IP或者ip段，如：.　　
iptables -I INPUT -s . -j DROP
iptables -I INPUT -s ./ -j DROP
#禁止从某个主机ssh远程访问登陆到本机，如123..　　
iptable -t filter -A INPUT -s . -p tcp --dport  -j DROP　　
#备份mysql数据库
mysqldump -u 用户名 -p 密码 数据库名 > back.sql　　　　
mysqldump --all-databases > bak.sql　　　　　　
#还原mysql数据库
mysql -u 用户名 -p 密码 数据库名 < bak.sql　　
find / *.php -perm  　　 　　
awk -F:  /etc/passwd　　　　
crontab -l　　　　
#检测所有的tcp连接数量及状态
netstat -ant|awk  |grep |sed -e  -e |sort|uniq -c|sort -rn

查看某个文件在哪里，例如php.ini

sudo find / -name 'php.ini'
